# 发布检查清单

发布一篇正式文章前后，按这份清单检查。

## 文章本身

- [ ] 文件放在 `posts/年份/`
- [ ] 文件名符合 `YYYY-MM-DD-slug.md`
- [ ] 标题、日期、标签、状态已经填写
- [ ] 正文已经完成，示例和链接可用
- [ ] 图片和附件路径检查通过

## 仓库导航

- [ ] `index.md` 已添加文章入口
- [ ] `archive.md` 已添加年份归档
- [ ] `tags.md` 已补充对应标签
- [ ] 如果属于系列文章，已更新 `series/` 下对应导读

## 资源整理

- [ ] 图片已经放到 `assets/images/`
- [ ] 附件已经放到 `assets/files/`
- [ ] 文件名清晰，不使用无语义名称

## 发布后自查

- [ ] 在 GitHub 页面点开文章检查排版
- [ ] 检查相对链接是否能正常跳转
- [ ] 检查代码块、标题层级和列表显示是否正常

## 备注

如果你发布频率比较高，可以每次复制这份模板到草稿里，作为发文前的最后一步。
